# P2P шахматы на C++ 

Студенческий проект для игры в шашки по локальной сети. 
Реализован на языке С++ с использованием библиотеки SFML для графического интерфейса. Для взаимодействия используется модель Peer-to-peer.

## Функционал
- Классическая игра в шашки между двумя игроками
- P2P соединение между игроками в локальной сети
- Использование TCP сокетов для надежной передачи данных
- Возможность настройки ip адресов и портов через конфигурационный файл
- Проверка корректности ходов противника
- Экраны
  - Ожидание подключения второго игрока
  - Игровое поле
  - Окончание игры (победа/поражение)
  - Некорректный ход
  - Потеря соединения

## Как запустить?

### Готовый исполняемый файл
1. Скачайте архив с последним релизом из раздела [Releases](https://github.com/LaGGgggg/checkers/releases)
2. Запустите `main.exe` (Windows) или `main` (Linux/Mac OS)

### Самостоятельная сборка через [CMake](https://cmake.org/)

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/LaGGgggg/checkers.git
   cd checkers
   ```

2. Настройте сборку:
   ```bash
   cmake -B build
   ```

3. Скомпилируйте проект:
    ```bash
    cmake --build build
    ```
    Для релизной сборки:
    ```bash
    cmake --build build --config Release
    ```

4. Запустите приложение:
    ```bash
    ./build/bin/main.exe  # Windows
    ./build/bin/main      # Linux/Mac OS
    ```

## Как играть?

### Требования
- Два игрока, находящихся в одной локальной сети или имеющих возможность подключиться друг к другу по ip

### Настройка
Создайте файл `config.txt` в той же директории, что и `main.exe/main` и установите ip другого игрока в формате:
```txt
ip=141.33.4.12
```

### Настройки firewall для Windows

Для связи между двумя игроками необходимо добавить разрешение в firewall Windows:

Откройте powershell **от имени администратора** и выполните
(подставьте ip другого игрока и измените порты, если вы их меняли):
```powershell
New-NetFirewallRule `
  -DisplayName "Checkers" `
  -Direction Inbound `
  -Action Allow `
  -Protocol TCP `
  -LocalPort 9001,8012 `
  -RemoteAddress 141.33.4.12

New-NetFirewallRule `
  -DisplayName "Checkers" `
  -Direction Outbound `
  -Action Allow `
  -Protocol TCP `
  -LocalPort 9001,8012 `
  -RemoteAddress 141.33.4.12
```

*Для удаления всех установленных правил для приложения выполните:*
```powershell
Remove-NetFirewallRule -DisplayName "Checkers"
```

### Запуск игры
1. Один игрок должен запустить приложение с аргументом командной строки `client`, другой - без:
   ```bash
   ./main.exe client  # Игрок 1
   ./main.exe         # Игрок 2
   ```
   Приложение, запущенное без аргумента позволяет играть белыми шашками, с агркментом `client` - черными
   
3. После успешного подключения экран загрузки сменится на игровое поле, после чего игра может быть начата

## Дополнительные возможности

### Смена шрифта
Шрифт находится в файле `font_data.cpp` в виде массива байтов.
Для генерации такого файла можно использовать `xxd` (подкомпонент `vim`):
```bash
xxd -i main_font.ttf > font_data.cpp
```

### Смена портов
По умолчанию используются порты `9001` и `8012`.
При необходимости можете изменить значения в `config.txt`:
```txt
port_1=9001
port_2=8012
```
Для корректной работы порты у обоих игроков должны совпадать.
